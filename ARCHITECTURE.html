🎯 한 줄 요약

LawOS는 변호사를 "서류 자동화 도구 사용자"에서 "지능형 전투사령관"으로 만드는 플랫폼이다.

단순 업무 자동화(Clio·Harvey)가 아니라, 온톨로지(사건 구조) × 이벤트(새 정보) × 전술 카탈로그로 지금 이 순간 가장 효과적인 행동을 제안하는 팔란티어 Gotham의 법률판.

🏗️ 6단계 아키텍처 레이어

각 층은 독립적으로 확장 가능. 위로 갈수록 지능이 높아짐.

Layer 6. Strategic Recommendation NEW

현재 상태 + 새 정보를 보고 "지금 이 상황에서 가장 효과적인 행동"을 자동 제안.

규칙엔진 (스테이지 × 서류 × 이벤트 × 약점) → 우선순위별 추천 카드

Layer 5. Tactical Playbook NEW

법정 전술 카탈로그. 15개 전술 각자 법적 근거·성공률·리스크 명시.

이혼 5 · 개인회생 3 · 형사 3 · 민사 3 · 상속/노동 확장 여지

Layer 4. Action Catalog

변호사가 실행할 수 있는 모든 Action의 선언적 정의.

입력 검증 · 권한 체크 · 감사 로그 · 사이드 이펙트 선언

Layer 3. Workflow Engine

사건 유형별 템플릿. 스테이지 진행 + 서류 체크리스트 + 자동 전환 규칙.

개인회생 템플릿: 6 스테이지 + 20 서류 + 14 액션

Layer 2. Ontology

데이터의 의미 구조. Client → Case → Ticket → Event → Attachment → Counterparty 그래프.

DB 테이블은 11개. 모든 테이블이 workspace 격리 + RLS 적용.

Layer 1. Data Sources

원시 데이터 입구. 다채널 입력으로 구조화 데이터로 전환.

📋 텍스트 붙여넣기 · 🎙 실시간 음성(STT) · 📦 벌크 텍스트 · 📎 파일 업로드 · 🔗 고객 포털 · 📧 이메일

🎙 상담 코파일럿 완료

실시간 STT (리턴제로 RTZR) → 전사 → Claude 분석 → 할일 자동 추출 → Triage 삽입. 고객 컨텍스트(기존 사건·할일) 자동 주입.

📋 텍스트 분석 완료

이메일/카톡/통화 녹취 붙여넣기 → LLM 구조화 추출 → 3가지 타입(promise / doc_request / follow_up) 티켓 자동 생성.

📦 벌크 온보딩 완료

기존 사무소 데이터 자유 형식으로 붙여넣기 → 고객 N명 + 사건 + 진행이력 + 할일 한 번에 파싱 → 프리뷰 → 일괄 커밋.

🧠 자연어 검색 (Cmd+K) 완료

"이번 주 마감 지난 할일", "김민수 개인회생 중 진행중" → Claude가 구조화 필터로 변환 → 결정론적 조회 (RAG 아님).

🗂 개인회생 워크플로우 완료

6 스테이지(수임→서류→조사→채권자→제출→변제) × 20 서류 × 14 Action. 진행률 / 다음 행동 자동.

⚖️ 법원별 전략 힌트 완료

서울회생법원(매우 엄격 180일) / 수원(엄격 150일) / 부산 등 자동 감지. 법원별 보정명령 빈출사유 · 대응 팁.

🔍 편파변제 탐지 (AI) 완료

통장내역 붙여넣기 → Claude가 6개월 내 의심 거래 추출. 법정 쟁점 대비 소명자료 사전 구축.

🧮 변제계획 시뮬 완료

소득/가족/채무 → 법정 기준 월 변제액 계산 + AI 경로 추천(회생/파산/워크아웃/프리워크아웃).

📄 PDF 자동 생성 완료

수임계약서 · 내용증명(개인회생 예정 통보) 한국어 PDF 생성 → Storage 저장 → 첨부 자동 연결.

🔗 고객 포털 완료

변호사가 14일 유효 magic link 생성 → 고객이 로그인 없이 서류 업로드. 사건 자동 연결.

🤖 AI 서류 자동 분류 완료

업로드 파일 → Claude가 20종 서류 중 매칭 (confidence ≥ 70%) → workflow 상태 자동 업데이트.

✨ AI 초안 생성 완료

In Progress 티켓 → 고객 컨텍스트 주입 → Claude가 이메일 초안 작성 → Review & Send 컬럼 이동 → 편집 → 실제 발송 (Resend).

📅 자동 리마인더 (Cron) 완료

매일 09:00 KST 일일 다이제스트. 09:30 KST 미수령 서류(3일+) 자동 재요청 이메일.

⚔️ 전술 플레이북 NEW

15개 합법 전술(본인신문/가압류/재산개시/내용증명 등) × 상대방 약점 매칭 → 성공률·리스크 표시.

👥 팀 + 권한 완료

owner/admin/member 역할. 이메일 초대 자동. 사건별 visibility(workspace/assigned_only/owner_only) 변호사법 §26 대응.

📊 대시보드 + 캘린더 완료

KPI 4개(활성 사건/고객/팀원/할일) + 월별 수임·종결 차트 + 최근 활동. /calendar 월별 그리드.

⚔️ 전략 자동화 (Palantir Combat 층)

단순 업무 자동화는 Clio/EvenUp도 함. LawOS의 진짜 해자는 승리 전략 제안.

예시: "공개 기피 회장 이혼" 케이스

1. 사건: 김○○ vs ○○그룹 회장 이혼 (재산분할 50억) 2. 상대방 프로필 생성 (의뢰인 동의 서명 기록 필수) 3. 약점 추가: - "공개석상 기피" (출처: 언론 기사 3건 · public_record) - "평판 민감" (출처: 공인탐정 보고서 · detective) 4. 시스템 자동 매칭: ┌─────────────────────────────────────────────┐ │ ⚔️ 공격 · 본인신문 신청 │ │ ● 약점 매칭 2개 │ │ 성공률 55% · 리스크 중간 │ │ 법적 근거: 민사소송법 §367, §150 │ │ 예상 효과: 출두 거부 → 의제자백 / │ │ 출두 → 진술 흐트러짐 + 합의 제안 │ │ [채택] [상세] │ ├─────────────────────────────────────────────┤ │ ⚔️ 공격 · 재산명시·조회 신청 (가사법 §48-2) │ │ 성공률 70% · 리스크 낮음 │ ├─────────────────────────────────────────────┤ │ ⚔️ 공격 · 재산분할 가압류 │ │ 성공률 65% · 리스크 중간 │ └─────────────────────────────────────────────┘ 5. 전술 채택 → 계획→실행→완료 상태 추적 6. 상대 대응 들어오면 Ontology 업데이트 → 새 전술 자동 재계산

윤리 가드레일 (아키텍처에 내장)

10가지 가드레일: 잘못 설계하면 "스토킹 앱" 낙인. 한 번만 걸려도 회사 끝. 타협 불가.
  1. 상대방 프로필 생성 시 의뢰인 동의 필수 — DB에 consent_recorded + scope 기록
  2. 약점에 출처 태그 필수 — public_record / interview / detective / news / sns_public / client_provided
  3. 불법 출처 서버 차단 — wiretap / hacking / stalking / bribery 값 거부
  4. 전술 카탈로그에 불법 전술 아예 없음 — 코드 레벨에서 배제
  5. 고위험 전술 경고 배지 — risk_level=high 시 빨간 박스 + 법적 리스크 명시
  6. 감사 로그 (누가 언제 생성/접근)
  7. 사건 종결 + 법정 보관기간 경과 시 자동 삭제 (V2)
  8. 변호사법 §26 비밀유지 명시 (UI 배너)
  9. 개인정보보호법 §15, §17 준수 안내
  10. 변호사 최종 책임 — 모든 전술 카드에 "채택 전 판단 필수"
