1. 설계 원칙

채무자회생법(DRBA) 및 실무 운용을 기준으로, 엔티티·관계·파생계산·제약·상태를 분리한다. 모든 계산과 분기는 엔티티 속성에서 유도된다. 이 온톨로지는 다음 네 표면에서 공통 데이터 모델로 쓰인다.

표면용도
챗봇 인테이크엔티티 속성을 질문 시나리오로 변환. 리스크 플래그 자동 점수화.
시뮬레이터§5 계산식을 슬라이더·입력으로 노출. 월변제액·변제율 실시간 산출.
사건관리 DB엔티티 그대로 테이블. Stage는 상태머신 테이블로 분리.
콘텐츠 택소노미Debt 유형·Asset 유형·Stage·리스크 플래그를 블로그/YouTube 카테고리로 매핑.

2. 엔티티 카탈로그 (13)

네 범주로 분류한다: 프로필 · 주체 (5) 사건 도메인 (3) 프로세스 · 이벤트 (4) 행위자 (1 타입 · 4 인스턴스)

2.1 Debtor · 채무자 (주체) v0.2 수정

사건의 주체. 프로필·사건·프로세스 모든 엔티티가 Debtor를 직접 또는 간접으로 참조한다.
식별name, rrn(암호화), age, gender, contact
주거residence_type(자가/전세/월세), deposit_amount(전세·월세 보증금액) 신규
직업job_types[](배열 — 근로·사업·프리랜서·무직 복합 허용) 근로+사업 복합 허용
적격성 플래그has_prior_discharge_within_5y(면책 후 5년 이내 재신청 불가) 5년으로 수정, has_regular_income, is_business_income_earner, unsecured_debt_cap_ok(≤10억), secured_debt_cap_ok(≤15억)
리스크 플래그has_tax_arrears(국세체납), has_insurance_arrears(4대보험 체납), has_criminal_case(형사사건) — 실질 리스크는 이 3개뿐. 일반 민사소송은 무방 재정의. preferential_transfer_risk, fraudulent_transfer_risk, has_guarantor, recent_loan_within_3m(최근 3개월 내 대출 — 사기죄 피소 위험) 신규
단축계획 자격shortening_eligibility(Debtor 필드 — §2.7 참조) 신규: is_under_30, is_over_65, is_single_parent, has_2plus_minor_children, is_jeonse_fraud_victim, is_severely_disabled
공포마케팅 포인트 (실무) — 민사소송 자체는 개인회생에 장애가 되지 않으나, 소송이 진행 중이면 빨리 신청하는 것이 유리하다는 정직한 긴급성 소구 가능. 4대보험·조세·형사사건은 우선변제 채권으로 취급되므로 실제로 까다롭다.

2.2 Debt · 채무 / 채권 v0.2 수정

채권자별 채무 단위. 변제계획과 청산가치 계산의 핵심 입력.
유형general_unsecured(일반무담보) / secured(담보부) / priority(우선권) / tax(조세) / non_dischargeable(비면책·형사) / private_loan(사채)
속성creditor_name, creditor_type(형사사건 채권자 = 실질 리스크 범주) 재정의, principal, interest, overdue_interest, origin_date, last_payment_date, cause
담보has_collateral, collateral_asset_id — 담보 설정된 자산만 청산가치 계산 시 채권액 공제 범위 축소
상태is_in_collection, is_litigated, judgment_finalized, has_guarantor, statute_of_limitations_expired

2.3 Asset · 재산 v0.2 수정

청산가치 산정의 기본 단위. 담보·면제 공제 후 순가치를 누적한다.
유형real_estate / deposit / security_deposit / insurance_surrender 수임 후 확정 / vehicle / retirement / business_asset 수임 후 확정 / receivable
평가market_value, liquidation_value, exempt_amount(압류금지·면제), secured_claims_on_asset
파생net_value = liquidation_value − exempt_amount − secured_claims_on_asset
수집 전략보험해약금과 사업용자산은 상담 단계에서 정확 파악 불가. 수임 후 보험증권·사업장 실사로 확정. 예상치 못한 자산이 나오면 그때 걸러내는 것이 인테이크 효율 측면에서 유리. 신규

2.4 Income · 소득 v0.2 수정

유형earned(근로) / business(사업) / freelance / public_benefit. 한 명이 복수 유형 보유 가능.
속성monthly_amount(3~12개월 평균), is_regular, is_documented(원천·4대보험), basis — 세후 기준으로 통일 세후 고정
사업소득 전략신고금액이 부정확한 경우가 많음. 계좌내역을 법원이 판단하지만, 인테이크 시에는 의뢰인에게 유리한 금액을 기재하도록 가이드하고 보정 단계에서 다투는 것이 실무. declared_for_intake, bank_evidence_amount 분리 기록. 신규

2.5 Dependent · 부양가족 v0.2 수정

가구원수는 최저생계비 산정의 결정 변수. 단순히 동거 여부로만 판단하지 않는다.
속성relation(배우자/자녀/부모/기타), age, is_cohabiting, has_own_income, is_minor
성년자녀 포함어린 성년자녀는 서울·수원 법원에서 일단 주장 가능. 기타 법원은 난이도 있으나 다툼 여지. young_adult_dependent_claim 플래그로 표시. 신규
가구원수 분할 산정배우자가 경제활동을 하면서 미성년 자녀가 있는 경우: 3인가구가 아닌 2인가구로 산정 (배우자와 자녀 1명, 본인과 자녀 1명으로 분할). household_size_logic = split 처리. 신규
가구원수 분할 예시 — 본인 + 경제활동 배우자 + 미성년 자녀 2명. 단순 합산 시 4인. 실무 산정은 2인 (본인 1 + 자녀 1). 최저생계비 계산이 크게 달라진다.

2.6 Case · 사건

법원에 접수된 사건 단위. 여러 Stage를 순차 경과하며, RepaymentPlan · Document · Interaction을 소유.
속성case_type(개인회생/일반회생/파산면책/소액영업소득자), court, case_number, trustee
일자filing_date, opening_date, approval_date, discharge_date
관계→ Stage(current), RepaymentPlan, Document[], Interaction[]

2.7 RepaymentPlan · 변제계획 v0.2 수정

변제기간plan_period_months ∈ {24, 36, 60} — 24개월 단축계획 신설 신규
단축계획 자격Debtor의 shortening_eligibility 필드에서 판정 (§2.1 참조):
· 만 30세 미만
· 만 65세 이상
· 한부모 가정
· 다자녀 가구 (미성년 자녀 2명 이상 양육)
· 전세사기 피해자
· 중증 장애인
기본 속성monthly_payment, total_payment, repayment_ratio, structure(equal/graduated)
검증liquidation_value_guaranteed — 청산가치보장원칙 충족 플래그

2.8 Document · 서류

신청서류개인회생신청서, 채권자목록, 재산목록, 수입지출목록, 진술서
증빙서류소득(원천·사업소득·통장), 재산(등기부·차량원부·보험증권), 채무(부채증명원·소장·독촉장), 가족(가족관계·주민등록)
속성doc_type, required, uploaded, verified

2.9 Stage · 절차단계 (상태머신) v0.2 수정

Case의 현재 단계. 선형이 아닌 루프·우회 허용 상태머신. 상세 흐름은 §4 참조.
단계상담 → 수임 → 서류준비 → 신청접수 → (보정 루프) → 개시결정 → 채권신고·조사 → 채권자집회 → 변제계획 인가 → 변제수행 → 면책 / 폐지 / 변경
기각 우회보정 단계에서 기각이 나올 경우 즉시항고 → 기각취소 → 개시결정 우회 경로 존재. 신규
속성entry_date, exit_date, required_actions[], blocking_issues[], responsible_party

2.10 Interaction · 상호작용

법원·대리인·채무자·회생위원·채권자 간 단일 소통 단위. 보정 루프가 1차·2차·3차 반복되는 단위이기도 하다.
유형correction_recommendation(보정권고) / opinion_request(의견조회) / status_report(사정보고) / objection(이의) / response(답변) / immediate_appeal(즉시항고) 신규
속성iteration_number, initiator, recipient, due_date, response_date, items[], status(pending/submitted/accepted/rejected)

2.11 CourtOrder · 법원 명령 v0.2 수정

유형correction_rec(보정권고) / dismissal(기각) 신규 / dismissal_revoked(기각취소) 신규 / opening_decision(개시결정) / plan_approval(인가결정) / termination(폐지결정) / modification_approval(변경결정) / discharge(면책결정)
속성order_type, issued_date, required_actions[], deadline, appealable

2.12 RepaymentEvent · 변제 이벤트

변제수행 기간(24·36·60개월) 동안 매월 발생하는 이벤트 로그. 회생위원·채무자 주도.
유형payment_completed(납입완료) / payment_delayed(납입지연) / income_change(소득변동) / modification_request(변경신청) / termination_warning(폐지경고)
속성event_date, amount, reason, evidence, triggered_actions[]

2.13 Actor · 행위자 v0.2 수정

Interaction·RepaymentEvent·Stage 전이에서 책임과 마감을 가지는 당사자. 개인회생위원 = 파산관재인 역할 겸함.
타입debtor(채무자) / attorney(대리인) / court(재판부) / trustee(개인회생위원 ≡ 파산관재인 역할 겸) 병기 / creditor(채권자)
속성responsibilities[], contact_channel(전자소송/카톡/서면/대면)

4.1 단계별 판례·소통 매트릭스

단계판례소통핵심 이슈
상담○○유사 사례 조회, 의뢰인 초기 진단
수임○위임계약 · 수임료 · 추심 제한 효과 고지
서류준비○채무자↔대리인 반복 소통. 증빙 수집의 완성도가 후속 보정 빈도를 좌우.
신청접수인지대·송달료 납부. 중지·금지명령 병행 가능.
보정 루프 (×N)○○법원↔대리인. 1차는 서류 보완이 대부분, 2·3차로 갈수록 실체적 쟁점(가용소득 산정·재산 평가).
기각 신규○법원 실수 또는 보정 과정에서의 마찰로 간혹 발생.
즉시항고 신규○○기각 결정에 대한 항고. 판례 근거 중요.
기각취소 신규○상급심의 취소 결정 후 개시결정으로 복귀.
개시결정○결정 송달 수령. 채권자에 대한 추심 본격 금지.
채권신고·조사○채권자↔법원 신고. 대리인이 이의 대응.
채권자집회○○이의 대응 논리에 판례 필요.
변제계획 인가○인가 요건 해석에 판례 활용.
변제수행○○회생위원↔채무자 월 단위 소통. 변경 시 판례 근거 필요.
면책○면책 결정 통지.
폐지 (실패)○○폐지 방어 논리에 판례 필수.

5. 파생 계산 체인

3개 입력 그룹이 4단계 파생을 거쳐 월변제액·총변제액·변제율을 산출한다. v0.2에서 제약 조건 재정의: 가용소득이 0 또는 음수라도 알바·취업예정·생활비 절감 주장으로 진행 가능. 변제기간 24개월 옵션 추가.

Diagram C · 파생 계산 DAG v0.2 (가용소득 제약 완화, 24개월 옵션)

5.1 계산 규칙 요약

단계공식비고
최저생계비중위소득(가구원수) × 60%가구원수는 분할산정 로직 적용 (§2.5)

6. 업무 규칙

  1. 적격요건 — 무담보 ≤ 10억, 담보 ≤ 15억. 정기적·확실한 수입. 지급불능 또는 그 우려.
  2. 변제기간 — 24개월(단축), 36개월(원칙), 60개월(연장) v0.2 확장. 청산가치 보장을 위해 최장 5년까지 연장.
  3. 청산가치보장원칙 — 총변제액은 청산가치 이상이어야 한다.
  4. 비면책채권 — 양육비, 벌금·과료·추징금, 고의 불법행위 손해배상, 일부 조세.
  5. 편파변제 / 사해행위 — 신청 전 6개월~1년 내 특정 채권자 변제·재산 처분은 부인 대상.
  6. 재신청 제한 — 개인회생 → 개인회생은 면책 후 5년 경과 요건 v0.2 수정 (7→5). 절차: 신청 → 개시 → 3년 변제 완료 → 면책. 면책 후 5년 내에는 회생신청 불가.
  7. 중지·금지명령 — 신청과 동시 신청 가능. 추심·경매 정지 효과.
  8. 시효 관리 — 시효소멸 채권은 채권자목록 등재 방식에 따라 변제 대상에서 전략적 처리 가능.

8. 적용 표면

8.1 챗봇 인테이크

엔티티의 속성을 질문 시나리오로 치환한다. Debtor·Income·Asset·Debt 의 필수 필드를 순서대로 수집하고, 리스크 플래그(v0.2 재정비된 7개)는 조건 트리거로 자동 점수화한다. §7.1 분기 로직을 내부에서 돌려 등급을 산정. 24개월 단축계획 자격 판정은 Debtor 수집 단계에서 함께 수행.

8.2 시뮬레이터

§5 계산 체인을 슬라이더/입력 UI로 노출. 세후 월소득·자산·채무를 입력하면 월변제액·변제율·총변제액이 실시간 업데이트. 변제기간은 24/36/60 선택 가능. 청산가치보장원칙은 경고로 시각화. 가용소득 0 이하도 진행 가능 케이스를 별도 안내.

8.3 사건관리 DB

엔티티를 그대로 테이블로. Stage는 별도 상태머신 테이블 + 전이 이력 로그. 기각 → 즉시항고 → 기각취소 우회 경로는 특수 전이로 기록. Interaction·RepaymentEvent는 이벤트 로그로 누적되어 "오늘의 할 일" 자동 생성 근거가 된다.

8.4 콘텐츠 택소노미

Debt.class · Asset.type · Stage · 리스크 플래그를 블로그/YouTube 카테고리 트리로 매핑. v0.2 기준 예시 카테고리: "3개월 내 대출 후 개인회생 (사기죄 리스크)", "면책 후 5년 — 재신청은 언제부터?", "가용소득 0이어도 개인회생 가능한 케이스", "24개월 단축계획 자격 6가지".

9. 미포함 / 확장 여지

이 v0.2에 의도적으로 포함하지 않은 네 개 영역. 실무 투입 전 단계적으로 확장.

  • 데이터 소스 레이어 — 판례 DB (대법원 판례공보, 케이스노트, 로앤비), 홈택스 소득증빙 연동, KCB·NICE 신용정보, 등기부·차량원부·사건조회 시스템.
  • Actor × Stage 책임 매트릭스 — 업무플로우 역산의 마지막 퍼즐. 각 단계에서 "누가 무엇을 언제까지" 해야 하는지를 Actor별로 명시.
  • 보정 루프 실증 데이터 — 1차·2차·3차 보정에서 실제로 어떤 항목이 지적되는지. 양홍수 변호사 실무 케이스 2~3건 리버스로 쟁점 패턴화.
  • 삼각관계 메타뷰 — A(구조)·B(시간)·C(계산) 다이어그램이 실행 중 서로 어떻게 업데이트하는지.
